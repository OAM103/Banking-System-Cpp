# Banking System (C++)

## Описание проекта

Данный проект представляет собой упрощённую банковскую систему, реализованную на языке C++ в учебных целях.

Программа позволяет создавать счета различных типов, выполнять денежные переводы между ними, хранить историю операций и сохранять данные в файл для последующей загрузки.

Проект разрабатывался как практическая работа по изучению объектно-ориентированного программирования, стандартной библиотеки STL и современных возможностей языка C++.

---

## Цель проекта

Основной целью проекта являлось получение практического опыта разработки объектно-ориентированных программ на языке C++ и освоение современных инструментов стандартной библиотеки языка.

---

## Реализованный функционал

### Счета

Поддерживаются два типа банковских счетов:

* **DebitAccount** — дебетовый счёт, не допускающий уход в минус.
* **CreditAccount** — кредитный счёт с заданным кредитным лимитом.

Каждый счёт содержит:

* уникальный идентификатор (ID);
* имя владельца;
* текущий баланс.

---

### Банковские операции

Реализованы следующие возможности:

* создание дебетовых счетов;
* создание кредитных счетов;
* поиск счёта по ID;
* удаление счёта;
* пополнение счёта;
* снятие средств;
* перевод средств между счетами;
* ведение истории переводов.

---

### Работа с памятью

Для хранения объектов счетов используются умные указатели:

```cpp
std::unique_ptr<Account>
```

Это позволяет автоматически освобождать память и избегать утечек ресурсов.

---

### Использование STL

В проекте применяются контейнеры стандартной библиотеки:

```cpp
std::unordered_map<int, std::unique_ptr<Account>>
```

для хранения счетов по идентификатору.

```cpp
std::vector<Transaction>
```

для хранения истории переводов.

---

### Работа с файлами

Реализовано сохранение данных в текстовый файл:

```cpp
saveToFile(...)
```

и восстановление данных из файла:

```cpp
loadFromFile(...)
```

Поддерживается сохранение как дебетовых, так и кредитных счетов.

---

## Что не реализовано

В рамках учебного проекта не были реализованы:

* защита от повреждения файла данных;
* полноценная обработка всех возможных ошибок ввода;
* сохранение истории переводов в файл;
* пользовательское меню;
* авторизация пользователей;
* многопоточность;
* база данных;
* тестирование проекта.

---
